# User Journey - Project Management
A reprsentative user journey, for proejct managemnet, to help us hash out key features.

1. Start in an empty workspace
2. Create your entity hierarchy: Members, Teams, Projects, Tasks, 
3. Create records for all your team members, all your teams, and link them together
5. Create a bunch of projects
6. For each project, create a bunch of tasks
7. Create a list-view, to start organizing tasks in a backlog
8. Work out of this list-view in backlog refinement
   - NOTE: How to do discussions? A "threads" field type? 
9. Assign team members to tasks, and let them update statuses, move things around, etc. 
10. View tasks moving from left-to-right on a kanban board, or something similar
   - NOTE: How to do history? A "history" field type?
11. Create differnet dashboarding views, like a 'team velocity' dashboard, or 'my upcoming tasks'
